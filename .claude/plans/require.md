# 스케치 게임 요구사항

## 개요
여러 사람이 함께 즐기는 실시간 웹 기반 스케치 전달 게임 (Gartic Phone 유사)

---

## 방 시스템
- 사설 방 생성 가능 (방 코드/링크로 입장)
- 방 인원: 최소 1명, 최대 8명
- 방장(호스트)이 게임 시작 권한 보유
- 닉네임으로 참가

---

## 게임 흐름

### 1단계: 주제 입력
- 각 참가자가 자신만의 주제(단어/문장)를 입력
- 모든 참가자가 입력 완료 시 다음 단계로 진행

### 2단계: 주제 전달 & 반복 턴
- 자신의 주제를 옆사람(다음 순서)에게 넘김
- 모든 참가자가 동시에 동일한 작업 수행
- 한 명의 주제 체인을 전체 참가자가 실시간으로 볼 수 있어야 함
- 턴마다 교대로 반복:
  - **그림 턴**: 받은 주제(텍스트)를 보고 그림 그리기
  - **추측 턴**: 받은 그림을 보고 주제 텍스트 적기
- 턴 제한 시간 존재 (그림: 60~90초, 추측: 30초 등)

### 3단계: 게임 종료 조건
- 각 주제가 인원 수만큼의 턴을 거쳐 원래 주인에게 돌아오면 종료
- 예) 4명 → 각 주제 4턴 (주제→그림→추측→그림 또는 주제→그림→추측→그림)

### 4단계: 결과 공개
- 모든 참가자가 각 주제의 전체 체인을 함께 감상
- 체인: 원래 주제 → 그림 → 추측 → 그림 → ... → 최종 결과
- 한 주제씩 순서대로 공개 (슬라이드 형태)

---

## 기능 요구사항

### 필수 기능
- 방 생성 / 입장 (코드 기반)
- 실시간 플레이어 목록 표시
- 그림 그리기 캔버스 (펜, 지우개, 색상 선택, 굵기 조절, 전체 지우기)
- 텍스트 입력 (주제 입력 / 추측)
- 턴 타이머
- 결과 공개 화면

### 부가 기능 (선택)
- 게임 설정 (턴 시간 커스텀, 최대 인원)
- 그림 저장/공유
- 반응형 (모바일 대응)

---

## 비기능 요구사항
- 실시간 동기화 (모든 참가자 동시 진행)
- 낮은 레이턴시
- 별도 회원가입 없이 바로 플레이
